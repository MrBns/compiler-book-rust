We have built all four phases of our Pico compiler:

1. ✅ **Lexer** — turns text into tokens
2. ✅ **Parser** — turns tokens into an AST
3. ✅ **Semantic checker** — catches errors in the AST
4. ✅ **Code generator** — turns the AST into TypeScript

Now let's connect them all in `main.rs`.

## The Complete main.rs

Replace everything in `src/main.rs` with this:

```rust
// main.rs — The entry point of the Pico compiler
//
// This file glues all phases together:
//   1. Read the source file
//   2. Lex (tokenize)
//   3. Parse (build AST)
//   4. Semantic check (find errors)
//   5. Generate TypeScript

mod token;
mod lexer;
mod ast;
mod parser;
mod semantic;
mod codegen;

use std::env;
use std::fs;
use std::path::Path;

fn main() {
    let args: Vec<String> = env::args().collect();

    if args.len() < 2 {
        eprintln!("Usage: pico <source.pico>");
        std::process::exit(1);
    }

    let source_path = &args[1];

    // Step 1: Read the source file
    let source = match fs::read_to_string(source_path) {
        Ok(content) => content,
        Err(e) => {
            eprintln!("Error reading '{}': {}", source_path, e);
            std::process::exit(1);
        }
    };

    println!("Compiling: {}", source_path);

    // Step 2: Lex — turn source text into tokens
    let tokens = lexer::tokenize(&source);
    println!("  Lexed {} tokens", tokens.len());

    // Step 3: Parse — turn tokens into AST
    let mut parser = parser::Parser::new(tokens);
    let program = parser.parse();
    println!("  Parsed {} top-level statements", program.len());

    // Step 4: Semantic check — find errors
    let mut checker = semantic::SemanticChecker::new();
    checker.check_program(&program);

    // If there are errors, print them all and stop
    if !checker.errors.is_empty() {
        eprintln!("\nSemantic errors found:\n");
        for error in &checker.errors {
            eprintln!("  - {}", error);
        }
        std::process::exit(1);
    }
    println!("  Semantic check passed");

    // Step 5: Generate TypeScript
    let typescript_code = codegen::generate(&program);

    // Step 6: Write the output file
    // Change the file extension: "hello.pico" -> "hello.ts"
    let output_path = Path::new(source_path).with_extension("ts");

    match fs::write(&output_path, &typescript_code) {
        Ok(_) => {
            println!("  Generated: {}", output_path.display());
            println!("\nCompilation successful!");
        }
        Err(e) => {
            eprintln!("Error writing output file: {}", e);
            std::process::exit(1);
        }
    }
}
```

## Running the Compiler

First, build the project:

```bash
cargo build
```

Create a test file `hello.pico`:

```txt
fn add(a, b) {
    return a + b;
}

let x = 10;
let y = 32;
let result = add(x, y);

if result > 40 {
    print("Big result!");
} else {
    print("Small result.");
}

print(result);
```

Now run the compiler:

```bash
cargo run -- hello.pico
```

You should see:

```txt
Compiling: hello.pico
  Lexed 47 tokens
  Parsed 5 top-level statements
  Semantic check passed
  Generated: hello.ts

Compilation successful!
```

And `hello.ts` will contain:

```typescript
// Generated by Pico Compiler
// Do not edit this file by hand.

function add(a: any, b: any): any {
    return (a + b);
}

const x = 10;
const y = 32;
const result = add(x, y);
if ((result > 40)) {
    console.log("Big result!");
} else {
    console.log("Small result.");
}
console.log(result);
```

If you have Node.js installed, run it:

```bash
npx ts-node hello.ts
```

Output:

```txt
Big result!
42
```

**Our Pico program compiled and ran!** 🎉

## What We Built

Look at what we created from scratch in Rust:

```txt
hello.pico
    (Lexer)    -> tokens
    (Parser)   -> AST
    (Semantic) -> checked AST
    (CodeGen)  -> TypeScript
hello.ts
```

A real working compiler, written entirely in Rust!

Summary of all files:

| File | Purpose |
| --- | --- |
| `src/token.rs` | Token type definitions |
| `src/lexer.rs` | Turns text into tokens |
| `src/ast.rs` | AST node types |
| `src/parser.rs` | Turns tokens into AST |
| `src/semantic.rs` | Checks for errors |
| `src/codegen.rs` | Generates TypeScript |
| `src/main.rs` | Connects everything |

## What You Can Add Next

Our Pico compiler is small on purpose. Here are ideas to make it bigger:

**Easy additions:**

- `while` loops — `while condition { ... }`
- Unary minus — `-x`
- Better error messages with line numbers

**Medium additions:**

- Arrays — `let arr = [1, 2, 3];`
- String operations — `len(s)`, string concatenation
- Multiple return values

**Hard additions:**

- Type checking — catch `"hello" + 5` as an error
- Import/export statements
- Generate Go code instead of TypeScript!
